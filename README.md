# Docker Practice 100 🐳🔥

A structured, hands-on repository containing **100 days of Docker practice**. This project is designed to help you learn Docker from basics to advanced concepts through daily tasks, real-world scenarios, and practical implementations.

---

## 📌 Overview

This repository follows a **day-wise learning approach**, where each folder represents a specific Docker concept, problem, or real-world use case.

Whether you're a beginner or preparing for DevOps roles, this repo provides a **practical roadmap to mastering Docker**.

---

## 📂 Repository Structure

* **Day-01 to Day-07** – Basics and foundational concepts
* **Day-08** – Installing Docker & starting services
* **Day-09** – Deploying Nginx container
* **Day-10** – Pulling Docker images
* **Day-11** – Managing Docker permissions
* **Day-12 & 13** – Creating Docker images from containers
* **Day-14** – Docker exec operations
* **Day-15 & 16** – Writing Dockerfiles
* **Day-17 & 23** – Docker networking
* **Day-18 & 26** – Port mapping
* **Day-19 & 28** – Docker Compose
* **Day-20 & 29** – Debugging Dockerfiles
* **Day-21 & 31** – Deploying applications in containers
* **Day-22 & 33** – Docker Python applications
* **Day-25** – Volume mapping
* **Day-27** – Image backup and transfer
* **Day-30** – Docker Compose troubleshooting
* **Day-32** – Docker Node.js application

---

## 🧠 What You Will Learn

* Docker installation & setup

## 🚀 Getting Started

1. Clone the repository:

   ```bash
   git clone https://github.com/abhijitray7810/Docker-Practice-100.git
   cd Docker-Practice-100
   ```

2. Navigate to any day’s folder:

   ```bash
   cd Day-09
   ```

3. Follow the instructions inside:

   * `README.md`
   * `Commands.md`
   * `Solution.md`
   * `Dockerfile` (if available)

---

## 🧪 Example

Run an Nginx container:

```bash id="ng1xrun"
docker run -d -p 80:80 nginx
```
